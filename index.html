15 Diffrent web devolpe words

Element

Element är en samling av alla taggar där taggar är det som identiferaroch definerar delar av din kod. Det finns en biblotek av olika taggar som kan användas, de vanligaste är: h1 (beteknar rubrik eller text typ ett), body (där all kod blir skrivet i), div (skilljer koder) och a ( används för länkar, kallas även ancker). Om man vill veta vad för Element en sida har kan man lätt hitta genom att kolla in genom inspecter.

Html

Hypertext markup language eller HTML kort, är det märkspråk som användas på hypertext eller själva denna hemsidan nu. Denna hemsida som massa andra använder sig av Html då det är standarden med HTTP för WWW. sidor.

ID

Id eller Id attribut är en specifik identitet till ett element. Jag har till exempelt använt mig av det på min snabb knapp. Det används för att kunna ändra en specifik h1 till exempel så att inte alla påverkas av ändringen.

Class

Class eller klasser är det kod som ger en group av element samma egenskaper eller stylesheet justeringar. Det är liknande Id fast skilljer sig då id bara håller sig till en specifk element.

Meta

Meta eller metadata, ger information om datorn, vem som har skrivit sidan och beskrivning och utseende av sidan. Man kan lätt hitta den vid toppen av koden under head taggen.

Responsive

Responsive, eller alltså en Responsive sida, är en sida som är anpassad för att kunna funka på flera enheter. Det kan till exempel vara att sidans bilder eller text justerar storlek för att se bra ut på en mobil skärm. Just nu används bibloteket bootstrap för att få denna justeringen att funka utan att behöva skriva in massa kod, men mananvänder sig av flexbox för att få det att funka.

http

http eller Hypertext transfer protocol, används för att transporta data från sidor till användare. Varje sida använder sig av http och Www. Där WWW får hemsidorna att köras på en online server så att alla har möjlighet att ta sig in på sidorna. datorn kommunicerar med http genom att skicka ut en http ruqest tag och en http response tag.

https

https eller Hypertext tarnsfer protocol secure, som i namnet är en säkrad variant av http. https gömmer information som till exempel log ins eller personnummer på sidor så att andra inte får tag på det. Det som vill gömmas krypteras och bara hosten eller ägaren har möjlighet att dekryptera det. SSL eller TLS är protokoller som får https att fungera. Enligt Herman så är det dyrt med SSL. :)

compressed data

Compressed data, då koden görs på ett sätt så att färre utrymme tars upp av filen. Till exempel så används Compressed data på bootstrap då man kan välja bootstrap.min istället för vanila versionen. bootstrap.min där min betyder mini vilket som sagt menar att datan är mer kompakt och mindre. Detta är bra för datorer eller hårdiskar som inte klarar av mycket utrymmes upptagning.

CDN

CDN eller Content delivery network, då man får sidor att ladda snabbare... mycket snabbare. Detta funkar genom fördelade nätverks servers som har olika positioner runt om jorden. Dessa servers bär på filer,HTML och javascript. Detta får användare från långa distanter att kunna snabbt få in informationen på en sida även om hosten bor på andra sidan jorden. CDN har även skydd mot DDOSer, vilket är cyber attacker mot servrar och nätverk. Så att till exempel hunden på bilden kan snabbt ladda in i sin favorit sida... DogHub !

Local bootstrap

Local bootstrap är den bootstrappingen som är lokalt som laddas ner till datorn istället för att vara använt online. Bootstrap är då en biblotek med massa kod som funkar som en preset för sin css. man kan såklart justera cssen på till exempel en h1 tag om man tycker att färgen på fonten inte passar. Fördel med att använda bootstrap är dens möjlighet att vara Responsive.

JQuery

JQuery är liknande bootstrap fast för javascript. Den är också ett biblotek som fixar preset åt din sida. Fördel med JQuery är dess kompakthet då långa linjer av kod förkortas med hjälp av enkel javascript kod.

Flexbox

Flexbox är inom HTML och klassas som en modell. Den ger dig möjlgiheten att skapa lådor som enkelt går att flytta runt på sidan. Fördelen med Flexbox är att slippa floats och positioning kod. [Justify.content , display: flexbox; osv]

Grid

Grid funkar på samma sätt som Flexbox, alltså en layout modell. Den får till ett grid eller rutnätt som gör det enklare att plasera till exempel bilder och text. Som flexbox så slipper man såklart floats och positioning med Grid.
